# ASSA ABLOY Real-Partner Benchmark Plan

This repository now targets the real ASSA ABLOY partner press-shop benchmark.

## Objective

Run an isolated local A/B benchmark comparing:

- Condition A: HumanEnerDIA without OVOS and without chatbot assistance.
- Condition B: HumanEnerDIA with OVOS and chatbot assistance.

The benchmark uses the imported ASSA ABLOY partner press-shop dataset and must not use the live production server.

## Required Evidence

- A/B measurement export from the Chrome measurement extension.
- KPI summary generated by `scripts/lab/summarize_ab_results.py`.
- Verification that the benchmark covers Monitoring, Analyses, and Documentation modules.
- Verification that answers are based on the real partner data and do not use simulated-factory assets.

## Execution Sequence

1. Clone the repository to a non-production Ubuntu machine.
2. Copy `.env.assaabloy.example` to `.env`.
3. Place the private partner ZIP under `data/raw/`.
4. Run `scripts/lab/bootstrap_assaabloy_lab.sh`.
5. Start the separate OVOS runtime from `../ovos-llm`.
6. Run `scripts/lab/verify_assaabloy_lab.sh --require-ovos`.
7. Install the Chrome extension from `pilot-measurement-extension/`.
8. Run Condition A and Condition B for the fixed ASSA ABLOY task set.
9. Export the raw measurement CSV.
10. Run `scripts/lab/summarize_ab_results.py` to generate KPI evidence.

## KPI Targets

- Operational-user effort reduction: at least `30%`.
- Technical-user intervention reduction: at least `25%`.
- DIA module coverage: Monitoring, Analyses, Documentation.
