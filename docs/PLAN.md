# HumanEnerDIA Simulated Pilot Execution Pack

This document is the entrypoint for the simulated pilot work approved by management.

The pack is organized to support four outputs:
- `Video A`: HumanEnerDIA without OVOS and without the chatbot
- `Video B`: HumanEnerDIA with OVOS and chatbot support
- KPI comparison table
- short methodology and evidence pack

## Start Here
- Freeze the pilot identity in [factory-profile.md](simulated-pilot/factory-profile.md).
- Use the fixed task list in [task-set.md](simulated-pilot/task-set.md).
- Prepare the environment with [demo-environment-prep.md](simulated-pilot/demo-environment-prep.md).
- Record with [video-script-a.md](simulated-pilot/video-script-a.md) and [video-script-b.md](simulated-pilot/video-script-b.md).
- Capture metrics in [kpi-measurement-sheet.csv](simulated-pilot/kpi-measurement-sheet.csv).
- Validate the run with [rehearsal-checklist.md](simulated-pilot/rehearsal-checklist.md).
- Package the final evidence with [final-pack-checklist.md](simulated-pilot/final-pack-checklist.md).

## Frozen Decisions
- Simulated factory profile: Romanian automotive/plastics-oriented manufacturing facility.
- Personas:
  - operational user: facility / production manager
  - technical user: energy / maintenance / automation engineer
- Conditions:
  - `A`: no OVOS, no chatbot
  - `B`: OVOS + chatbot enabled
- Proposal KPIs:
  - `30%` reduction in operational-user effort
  - `25%` reduction in technical-user intervention need
  - proof of Monitoring, Analyses, and Documentation modules

## Suggested Weekly Sequence
1. Day 1: freeze factory profile, personas, KPI sheet, and task set.
2. Day 2: align data/view setup and enable manual-mode recording controls.
3. Day 3: rehearse the exact A/B scripts and trim weak prompts.
4. Day 4: run the final dry run and freeze the prompts.
5. Day 5: record Video A and Video B.
6. Day 6: complete KPI table and methodology note.
7. Day 7: assemble the final evidence pack.
